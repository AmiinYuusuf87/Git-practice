A repository (repo) is simply a project folder that Git tracks and monitors.

Git stores the entire project history inside a hidden folder named .git.

Never delete or edit the .git folder, or you will lose your history.

Git organizes your project workflow into three distinct, highly functional areas.

The Working Directory is where you edit and manage your actual local files.

The Staging Area is a temporary waiting room or shopping cart for changes.

The Repository is the permanent vault where saved commits are stored forever.

You run git add to move chosen changes from Working Directory to Staging.

You run git commit to permanently save staged changes to the Repository.

A commit is a permanent, unique snapshot of your project at that moment.

Every commit has a unique ID, an author, a timestamp, and a message.

Always write clear, present-tense commit messages describing what actually changed.

Run git init once to turn any local folder into a Git repository.

Run git status frequently to see which files are modified, staged, or untracked.

Run git diff to see line-by-line changes in your files before staging them.
